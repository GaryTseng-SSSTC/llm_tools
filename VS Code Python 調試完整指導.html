開始學習 查看配置

1. 安裝必要的擴展

開始調試之前，首先需要確保已安裝 Python 擴展，這是進行 Python 調試的必要條件

Python 擴展安裝

1

打開擴展市場

在 VS Code 中，前往側邊欄的擴展視圖（Extensions）

2

搜尋並安裝

搜尋並安裝 Python 擴展（由 Microsoft 提供）

VS Code - 擴展

2. 設定調試配置

在 VS Code 中，調試配置存儲在 launch.json 文件中，讓我們來創建和配置這個關鍵文件

配置步驟

1

打開運行和調試視圖

在側邊欄找到並點擊 "運行和調試" 視圖

2

創建 launch.json 文件

點擊 "創建一個 launch.json 文件" 的鏈接

3

選擇 Python 調試器

從選項中選擇 Python 作為調試器類型

launch.json 
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Python: 當前文件",
      "type": "python",
      "request": "launch",
      "program": "${file}",
      "console": "integratedTerminal",
      "justMyCode": true
    }
  ]
}

配置說明

VS Code 將自動生成基本的配置文件，您可以根據專案需要進行修改和擴展

3. 實際調試操作

學會設置斷點、開始調試，以及使用各種調試控制功能來排查和修復代碼問題

設置斷點

在您希望調試的代碼行旁邊點擊行號，將出現紅點，這表示您已設置了一個斷點。當程序執行到這一行時，將會暫停，讓您檢查變數的狀態。

💡 提示

斷點可以隨時添加或移除，只需再次點擊行號即可切換斷點狀態

開始調試

要開始調試，您可以使用以下方式：

按下 F5 鍵

最快的啟動方式

點擊開始調試按鈕

在運行和調試視圖中

調試控制快捷鍵

步過 (Step Over)

F10

跳過當前行，執行下一行

步入 (Step Into)

F11

進入函數內部

步出 (Step Out)

Shift + F11

返回到調用函數的地方

繼續 (Continue)

F5

繼續執行直到下一個斷點

變數檢查與監控

在調試過程中，您可以檢查變數值、監控表達式，並在調試控制台中執行代碼片段

變數面板

在調試過程中，變數面板會自動顯示當前作用域內所有變數的即時值，讓您輕鬆追蹤數據變化。

  • 本地變數
  • 全局變數
  • 函數參數

監視面板

在監視面板中添加自定義表達式，持續監控特定變數或計算結果的變化，幫助您發現問題。

  • 自定義表達式
  • 即時計算
  • 持續監控

調試控制台

在調試控制台中，您可以執行任意 Python 代碼片段，檢查變數狀態，或測試修復方案。

  • 執行代碼
  • 查看輸出
  • 互動式調試

進階調試技巧

掌握條件斷點和多配置支持等進階功能，讓您的調試工作更加高效和精準

條件斷點

右鍵單擊行號，選擇添加條件斷點，只有在滿足特定條件時才會觸發。這對於循環調試特別有用。

使用場景範例

  • i == 100 - 當循環變數達到特定值
  • error_count > 0 - 當錯誤數量超過閾值
  • data is None - 當數據為空值時

多配置支持

您可以在 launch.json 中添加多個配置，以便在不同情況下使用，例如不同的啟動參數或環境設置。

配置類型
  • • 當前文件調試
  • • 模組調試
  • • 遠程調試
  • • 帶參數啟動
多配置範例 
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Python: 當前文件",
      "type": "python",
      "request": "launch",
      "program": "${file}",
      "console": "integratedTerminal"
    },
    {
      "name": "Python: 模組",
      "type": "python",
      "request": "launch",
      "module": "myapp",
      "console": "integratedTerminal"
    },
    {
      "name": "Python: 帶參數",
      "type": "python",
      "request": "launch",
      "program": "${file}",
      "args": ["--debug", "--verbose"],
      "console": "integratedTerminal"
    }
  ]
}

最佳實踐建議

  • 合理設置斷點，避免在性能敏感的循環中設置過多斷點
  • 使用條件斷點來精確定位問題，減少不必要的停頓
  • 善用監視面板追蹤關鍵變數的變化趨勢
  • 為不同的專案需求建立專用的調試配置
