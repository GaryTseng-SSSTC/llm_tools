完整 n8n 工作流程配置

可直接匯入的 JSON 配置檔案，包含所有必要的節點與連接設定

工作流程節點說明

Webhook 觸發

接收 POST 請求，觸發整個工作流程。支援 JSON 格式的文本內容輸入。

OpenAI 節點（雙路徑）

並行執行兩個 GPT-4 Turbo 分析：一個生成 Mermaid 語法，一個生成結構化 JSON。

程式碼處理節點

清理 AI 輸出格式，驗證語法正確性，確保輸出品質。

回應節點（雙格式）

同時輸出精美的 HTML 網頁與純 JSON 資料，滿足不同整合需求。

重要提醒

匯入後請記得在兩個 OpenAI 節點中設定您的 API Key，並將 YOUR_OPENAI_CREDENTIAL_ID 替換為實際的憑證 ID。

使用說明與設定指南

詳細的安裝、配置與測試流程，讓您快速上手並開始使用

1

匯入與設定

匯入工作流程

  • • 複製完整 JSON 配置
  • • 在 n8n 點選「Import from Clipboard」
  • • 貼上 JSON 並確認匯入

設定 OpenAI 憑證

  • • 在兩個 OpenAI 節點設定 API Key
  • • 替換憑證 ID 為實際值
  • • 建議使用 gpt-4-turbo 模型
2

測試工作流程

啟動 Webhook

  • • 啟用工作流程
  • • 取得 Webhook URL
  • • 格式：/webhook/mindmap-generator

發送測試請求

curl -X POST [URL] \
-H "Content-Type: application/json" \
-d '{"content": "..."}'
3

查看結果

Mermaid 路徑

  • • 直接在瀏覽器開啟 URL
  • • 查看精美的互動式心智圖
  • • 支援深色/淺色模式

JSON 路徑

  • • 同時產生結構化 JSON
  • • 可匯入 Miro、Notion 等工具
  • • 支援進階 API 整合

進階客製化設定

調整 AI 提示詞

  • • 修改 OpenAI 節點的 messages 內容
  • • 調整層級深度與分支數量
  • • 自定義輸出風格參數

整合其他服務

  • • 新增 HTTP Request 節點
  • • 串接 Miro API、Notion API
  • • 參考社群整合範例

批次處理

  • • 加入 Split In Batches 節點
  • • 一次處理多個文件
  • • 設定 batchSize: 5 避免限制

效能優化建議

  • • Token 控制：設定 maxTokens: 2000 限制
  • • 快取機制：加入 Redis 或 SQLite 節點
  • • 錯誤處理：設定重試邏輯（3 次）
  • • 分段處理：長文本先用 Code 節點分段
