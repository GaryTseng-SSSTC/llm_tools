什麼是 uv？

uv 是由 Astral 團隊開發的現代化 Python 套件與專案管理器，旨在成為 Python 生態系的一站式解決方案

極致速度

採用 Rust 語言開發，並行處理與智慧快取，安裝速度比 pip 快 10-100 倍

一體化工具鏈

取代 pip、virtualenv、pip-tools、poetry 等多個工具，提供統一介面

跨平台一致性

uv.lock 鎖定檔案確保在不同作業系統和環境中的依賴一致性

核心功能

虛擬環境管理
依賴安裝與管理
專案初始化
套件建構與發布

新手入門教學

從零開始學習 uv，一步一步建立您的第一個專案

uv 工作流程

1

初始化專案

建立一個新的專案目錄並進入，然後使用 uv init 初始化專案結構。

建立專案目錄

mkdir my-awesome-project cd my-awesome-project

初始化專案

uv init
生成的專案結構
my-awesome-project/ ├── .venv/ # 虛擬環境 ├── pyproject.toml # 專案設定檔 ├── README.md # 專案說明 ├── .gitignore # Git 忽略檔案 └── main.py # 主程式
2

安裝依賴套件

使用 uv add 指令來添加專案需要的依賴套件。

添加基本套件

uv add requests fastapi

添加開發工具

uv add --dev pytest black ruff
自動完成的工作
  • • 更新 pyproject.toml 檔案
  • • 生成 uv.lock 鎖定檔案
  • • 自動安裝到虛擬環境
3

執行專案

使用 uv run 在虛擬環境中執行您的程式碼。

執行主程式

uv run python main.py

執行測試

uv run pytest
uv run 的優勢
  • • 自動偵測虛擬環境
  • • 執行前自動同步依賴
  • • 無需手動啟用環境
4

鎖定環境

使用 uv lock 生成鎖定檔案，確保團隊間環境一致性。

鎖定依賴

uv lock

同步環境

uv sync
uv.lock 檔案的重要性
  • • 記錄精確的依賴版本
  • • 跨平台相容性
  • • 確保可重現的環境

常見使用情境與指令範例

Python 版本管理

查看可用版本

uv python list

安裝 Python 版本

uv python install 3.12

設定專案預設版本

uv python pin 3.12

虛擬環境管理

建立虛擬環境

uv venv --python 3.12

臨時執行套件

uv run --with requests python script.py

清理快取

uv cache clean

依賴管理

從 requirements.txt 安裝

uv add -r requirements.txt

升級套件

uv add --upgrade requests

移除套件

uv remove package-name

工具執行

快速執行工具

uvx ruff check .

安裝全域工具

uv tool install black

升級工具

uv tool upgrade black
