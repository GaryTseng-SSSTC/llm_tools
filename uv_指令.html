常用 uv 指令與說明

完整的指令參考表格，包含功能說明、重點參數和實際範例

指令 功能定位 重點參數 對應腳本範例
uv run 在專案虛擬環境或臨時環境內執行程式、指令
--with <pkg>臨時安裝依賴
--python <ver> 指定 Python 版本
uv run --with cowsay main.pyuv run --python 3.13 script.py
uv init 初始化專案／單檔腳本，產生 .venv、pyproject.toml...
--lib 建立函式庫
--script <file> 單檔模式
uv init --lib mylib uv init --script tool.py --python 3.12
uv add 安裝套件並寫入 pyproject.toml、更新 uv.lock
--group dev 加入 dev 群組
-r <req.txt> 從檔案安裝
uv add requests fastapi uv add -r requirements.txt
uv remove 從專案移除套件並更新鎖檔
--group dev 僅移除特定群組
uv remove fastapi uv remove pytest --group dev
uv sync 依 uv.lock 內容同步環境
--locked 僅驗證不安裝
 uv sync # clone 專案後第一次執行
uv lock 解析並產生／更新 uv.lock
-U 升級全部依賴
uv lock # 產生鎖檔 uv lock -U # 升級並重新鎖定
uv export 匯出鎖檔為其他格式
--format pip / toml
 uv export --format pip > requirements.txt
uv tree 顯示依賴樹
--depth <n> 限制深度
 uv tree --depth 2
uvx 在一次性隔離環境中執行 CLI 工具
—
 uvx pycowsay "Hello uv!"
uv tool install 像 pipx 一樣全域安裝 CLI 工具
—
uv tool install ruff ruff --version
uv python 安裝、列舉、切換多版本 Python
install, list, pin
uv python install 3.12 uv python list uv python pin 3.12
uv venv 建立虛擬環境（極快速）
--python <ver> 指定版本
 uv venv --python 3.11
uv pip pip 相容介面（極快速）
與原生 pip 相同
uv pip install -r requirements.txt uv pip list
uv build 將專案打包成 wheel / sdist
--out-dir <dir>
uv build ls dist/
uv publish 發佈套件至 PyPI 或私有倉
--repository-url
--token
 uv publish --token $PYPI_TOKEN
uv cache 檢視、清理或停用全域快取
clean, dir, stats
uv cache stats uv cache clean
uv self update 升級 uv 本身
—
 uv self update

說明：

以上指令名稱與功能根據官方文件及操作範例彙整。腳本範例皆以 bash 命令列形式示範，可直接貼到終端機測試。

實用範例和最佳實踐

專案初始化

基本專案建立

# 建立新專案
uv init my-project
cd my-project

# 查看生成的檔案結構
ls -la
# .git/  .gitignore  .python-version  README.md  main.py  pyproject.toml

函式庫專案建立

# 建立函式庫專案
uv init --lib my-library
cd my-library

# 函式庫專案會有不同的結構
tree .
# ├── src/
# │   └── my_library/
# │       └── __init__.py
# └── pyproject.toml

單檔腳本專案

# 建立單檔腳本
uv init --script batch_processor.py --python 3.12

# 在現有腳本上加上依賴宣告
uv add --script batch_processor.py pandas numpy

# 執行腳本
uv run batch_processor.py input.csv

依賴管理

安裝依賴

# 安裝單個套件
uv add requests

# 安裝多個套件
uv add fastapi uvicorn pydantic

# 從 requirements.txt 安裝
uv add -r requirements.txt

# 指定版本
uv add "django>=4.0,<5.0"

開發依賴

# 安裝開發工具
uv add --group dev pytest black ruff mypy

# 安裝測試依賴
uv add --group test pytest-cov pytest-mock

# 只同步生產依賴
uv sync --no-group dev

版本管理

# 升級特定套件
uv sync --upgrade-package requests

# 升級所有套件
uv lock --upgrade

# 移除套件
uv remove pandas

# 查看依賴樹
uv tree --depth 2

環境管理

Python 版本管理

# 安裝 Python 版本
uv python install 3.12 3.11 3.10

# 列出已安裝版本
uv python list

# 設定專案 Python 版本
uv python pin 3.12

# 檢查當前版本
python --version

虛擬環境

# 建立虛擬環境
uv venv --python 3.12

# 指定虛擬環境位置
uv venv my-env --python 3.11

# 啟用虛擬環境（可選，uv run 會自動處理）
source .venv/bin/activate  # Linux/macOS
# .venv\Scripts\activate     # Windows

環境同步

# 首次克隆專案後同步
uv sync

# 驗證鎖檔但不安裝
uv sync --locked

# 只安裝生產依賴
uv sync --no-group dev

# 重新安裝所有依賴
uv sync --reinstall

# 以「可編輯模式」安裝當前目錄中的 Python 套件，
# 當前目錄必須包含 setup.py、setup.cfg 或 pyproject.toml 
# 等套件配置檔案。
uv pip install -e .

工具和腳本執行

全域工具安裝

# 安裝常用工具
uv tool install ruff black mypy

# 安裝特定版本
uv tool install "black==23.12.1"

# 列出已安裝工具
uv tool list

# 升級工具
uv tool upgrade black

臨時執行

# 臨時執行工具
uvx cowsay "Hello uv!"

# 指定套件版本
uvx --from "httpx==0.24.0" python -c "import httpx; print(httpx.__version__)"

# 臨時安裝依賴並執行
uv run --with requests --with beautifulsoup4 scraper.py

腳本執行

# 執行專案腳本
uv run main.py

# 帶參數執行
uv run manage.py runserver 8080

# 指定 Python 版本執行
uv run --python 3.11 script.py

# 在專案環境中執行任意命令
uv run pytest tests/

建構與發布

套件建構

# 建構套件
uv build

# 只建構 wheel
uv build --wheel

# 指定輸出目錄
uv build --out-dir build/

# 檢視建構結果
ls dist/

套件發布

# 發布到 PyPI
uv publish --token $PYPI_TOKEN

# 發布到 TestPyPI
uv publish --repository-url https://test.pypi.org/simple/ --token $TEST_PYPI_TOKEN

# 乾運行（不實際上傳）
uv publish --dry-run

CI/CD 整合

# GitHub Actions 範例
- name: Set up uv
  run: curl -LsSf https://astral.sh/uv/install.sh | sh

- name: Sync dependencies
  run: uv sync --locked

- name: Run tests
  run: uv run pytest

- name: Build package
  run: uv build

Docker 整合

基本 Dockerfile

FROM python:3.12-slim

# 安裝 uv
COPY --from=ghcr.io/astral-sh/uv:latest /uv /bin/uv

# 複製專案檔案
COPY . /app
WORKDIR /app

# 同步依賴
RUN uv sync --frozen

# 設定路徑
ENV PATH="/app/.venv/bin:$PATH"

CMD ["python", "main.py"]

多階段建構

# 建構階段
FROM python:3.12-slim as builder
COPY --from=ghcr.io/astral-sh/uv:latest /uv /bin/uv
COPY pyproject.toml uv.lock /app/
WORKDIR /app
RUN uv sync --frozen --no-dev

# 執行階段
FROM python:3.12-slim
COPY --from=builder /app/.venv /app/.venv
COPY . /app
WORKDIR /app
ENV PATH="/app/.venv/bin:$PATH"
CMD ["python", "main.py"]

最佳實踐建議

建議做法

  • 使用 uv sync 而非 uv run

    在 CI/CD 中明確同步環境

  • 版本控制 uv.lock

    確保團隊環境一致性

  • 使用依賴群組

    分離開發、測試、生產依賴

  • 定期更新依賴

    使用 uv lock --upgrade 保持最新

  • 使用 .python-version

    明確指定專案 Python 版本

避免做法

  • 手動編輯 uv.lock

    應該透過 uv 指令管理

  • 混用 pip 和 uv

    可能導致依賴關係不一致

  • 忽略 .gitignore

    .venv 應該被排除在版控外

  • 在生產環境安裝dev依賴

    使用 --no-group dev 避免

  • 跳過版本限制

    應該指定合理的版本範圍

快速提示

從其他工具遷移

  • • Poetry → uv: 轉換 pyproject.toml
  • • pip-tools → uv: 導入 requirements.txt
  • • Pipenv → uv: 使用 uv add 重建

效能優化

  • • 使用全域快取減少重複下載
  • • 在 Docker 中分層建構減少重建時間
  • • 使用 --locked 在 CI 中加速
