安裝與部署指南

選擇適合您的部署方式

Docker 部署方式

Docker 是最簡單且最受推薦的部署方式，提供一致的執行環境和簡化的管理。

連接本地 Ollama

適合已安裝 Ollama 的用戶

docker run -d -p 3000:8080 --add-host=host.docker.internal:host-gateway -v open-webui:/app/backend/data --name open-webui --restart always ghcr.io/open-webui/open-webui:main

捆綁 Ollama 一鍵部署 (CPU)

內建 Ollama，無需額外安裝

docker run -d -p 3000:8080 -v ollama:/root/.ollama -v open-webui:/app/backend/data --name open-webui --restart always ghcr.io/open-webui/open-webui:ollama

捆綁 Ollama (Nvidia GPU)

支援 GPU 加速，適合高效能需求

docker run -d -p 3000:8080 --gpus=all -v ollama:/root/.ollama -v open-webui:/app/backend/data --name open-webui --restart always ghcr.io/open-webui/open-webui:ollama

重要提示

-v open-webui:/app/backend/data參數至關重要，它會將 Open WebUI 的資料（如使用者帳號、對話紀錄）掛載到本地磁碟，防止容器重啟後資料遺失。

安裝步驟

  1. 1

    安裝 Docker

    前往 Docker 官網下載並安裝 Docker Desktop

  2. 2

    選擇部署指令

    根據您的需求選擇適當的 Docker 指令

  3. 3

    存取 WebUI

    在瀏覽器中開啟 http://localhost:3000

使用 Python 安裝 (uv)

適合不想使用 Docker 的用戶，官方推薦使用 uv 環境管理器進行安裝。

安裝 uv

macOS/Linux:

curl -LsSf https://astral.sh/uv/install.sh | sh

Windows (PowerShell):

powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

啟動 Open WebUI

macOS/Linux:

DATA_DIR=~/.open-webui uvx --python 3.11 open-webui@latest serve

Windows (PowerShell):

$env:DATA_DIR="C:\open-webui\data"; uvx --python 3.11 open-webui@latest serve

建議使用 Python 3.11

為獲得最佳兼容性和穩定性，建議使用 Python 3.11 版本進行安裝。

存取方式

安裝完成後，在瀏覽器中開啟 http://localhost:8080

  • • 更靈活的配置選項
  • • 無需 Docker 知識

使用者與權限管理

完善的多租戶管理系統

使用者角色

管理員 (Admin)

第一個註冊的帳戶將自動獲得管理員權限，擁有所有系統控制權

使用者 (User)

後續註冊的使用者預設為「待定」狀態，需要管理員批准

單使用者模式

設定 WEBUI_AUTH=False 停用登入頁面

權限分類

工作區權限

  • • 模型存取與管理
  • • 知識庫存取控制
  • • 提示詞模板管理
  • • 工具上傳與管理

聊天權限

  • • 檔案上傳權限
  • • 對話編輯/刪除
  • • 臨時聊天建立

功能權限

  • • 網路搜尋功能
  • • 圖像生成工具
  • • 程式碼解釋器

讓 AI 擁有專業知識領域的能力

文件 RAG

上傳 PDF、TXT、MD 等多種格式文件，建立專屬知識庫

支援多種文件格式
自動向量化處理
來源引用追蹤

網頁 RAG

直接輸入網頁連結，自動抓取並分析網頁內容

實時網頁內容抓取
智慧內容解析
無縫整合對話

YouTube RAG

貼上 YouTube 連結，自動提取影片字幕進行問答

自動字幕提取
影片內容問答
時間戳記定位

RAG 最佳實踐建議

上下文長度優化

建議將模型上下文長度調高至 8192 tokens 或以上，以容納更多檢索資訊

嵌入模型選擇

使用高品質的嵌入模型如 mxbai-embed-large，可顯著提升檢索準確性

混合搜尋技術

平台支援結合關鍵字搜尋 (BM25) 和向量搜尋的混合檢索方式

引用透明度

所有 RAG 回應都會附帶來源引用，確保資訊可追溯性和可信度
